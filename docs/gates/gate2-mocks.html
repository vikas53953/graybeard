The map — what your app is made of

🖥️ Screens (what you see) — 3 pages: browse, add recipe, shopping list · 🧠 Brain (the logic) — one small server that saves and searches · 🗄️ Memory (the data) — a single database file on your machine · 🔑 Keys (login) — SKIPPED for v1: it's only you, no accounts needed · 🏠 Home (hosting) — runs on your computer; going online is a v2 decision · 🔭 Watchtower (logs) — every error written to one file the AI reads before ever asking you · 🔒 Locks (security) — nothing leaves your machine in v1 · 🔌 Pipes (connections) — none; no outside services.

The principles — why it's arranged this way

✂️ One job per file — no file may grow into a monster that does everything (this is the #1 killer of AI-built apps). · 📍 One source of truth — your recipes live in exactly ONE place; no copies to drift apart. · 🎯 Simplest thing that works — no login system for an app only you use. · ➡️ One-direction flow — screen asks brain, brain asks memory; never sideways shortcuts. · 🔭 Explains its own failures — when it breaks, the log says where, so fixes don't start with guessing. · 🧩 Loose coupling — the shopping list can be deleted entirely without touching recipes.
